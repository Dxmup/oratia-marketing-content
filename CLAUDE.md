# book-to-linkedin

VRM Sales Library content factory. Turns sales books into LinkedIn posts, newsletters, and X/Twitter content targeted at vacation rental management (VRM) operators focused on owner acquisition.

**Owner:** Chad Hensel (chad.hensel@gmail.com) — VRM operator, founder of Oratia

---

## What this repo is

Each book in the queue gets processed through a multi-agent Content Factory pipeline, producing 3 LinkedIn posts, a newsletter, and a set of X posts. All content is scoped to VRM owner acquisition workflows and written in Chad's voice.

---

## Agent pipeline

| # | Agent | Output |
|---|---|---|
| 0 | Orchestrator | coordinates the run |
| 1 | Analyst | `nuggets.json` |
| 2 | Strategist | `strategy.json` |
| 3 | Writer | `linkedin/post_*.txt`, `newsletter/newsletter.txt` |
| 4 | Drafter | `draft_queue.json` |
| 5 | Editor | `editor_review.json` |
| 6 | Twitter Writer | `twitter/posts.json` |
| 7 | Scheduler | scheduling metadata |

Prompts for each agent live in `prompts/`.

After the Content Factory, content goes to **VRM Sharp** (`AGENT_INSTRUCTIONS.md`) for a sharpening pass before publishing.

## VRM Sharp rules (key points)

- Reads `voice_guide.md` and `style_samples.md` before touching any content
- Processes **one full book per run** — all pending files for one book slug, in one commit
- Generates `intro.txt` on first pass if it doesn't exist (uses `output/intro_templates.md`)
- Must verify `processing_log.json` is updated before committing (step 8 in `AGENT_INSTRUCTIONS.md`)
- Character limit (2,900) applies to `post_*.txt` and `intro.txt` only — not `newsletter.txt`
- Commit format: `Sharpen {book_slug}: linkedin/intro, post_1, post_2, post_3, newsletter, twitter/posts`
